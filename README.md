# IAM Governance & Zero Trust Security Policy Framework
## Project Overview

This repository contains a **strategic set of IAM Recommendations** and a **comprehensive Information Security Policy (ISP)** centered on Identity and Access Management. In a modern threat landscape, **identity is the new perimeter**. This project demonstrates the application of **Zero Trust** principles to protect confidential data from unauthorized internal and external access.

## Scenario

A company recently discovered critical vulnerabilities in the protection of its flagship product formula. An internal audit revealed that:

* All research lab employees had access to the formula, regardless of their specific role, violating the principle of **least privilege.**
* Only a basic username and password was needed to access the research lab.
* The formula's creator maintained full administrative rights, violating the principle of **Separation of Duties.**

The company engaged an **external GRC Analyst** to provide remediation recommendations and draft a formal IAM Policy to prevent a future similar scenario.

## Methodology

The documentation in this respository addresses many facets of identity and access management, but most notably mandating **identity-based perimeters**, implementing **Role-Based Access Control (RBAC)** to set permissions based on job role instead of departmental access, mandating **Multi-factor authentication** for everyone, and implementing **user access reviews**.
